These are all properties you use for web layout, but they can have different uses when it comes to print.

Respect Your Users' Ink

When you start a design project, you need to know the constraints: who's the audience, what media will the project appear in, what's the budget? When you're building a print stylesheet for a web page, you don't really know what those constraints are. Sure, you could target the stylesheet at high-end multi-color inkjet printers that can handle a variety of paper sizes, but most of your home audience is probably working with something a lot simpler, like a black-and-white 8.5 x 11 printer, and they're probably not printing your web page to frame and hang on the wall. 

It's another way of thinking about responsive design: responsive for the home-printer. This doesn't mean that you need to eliminate all color from the print CSS, but make sure that whatever color you use is going to look ok in gray as well. And most importantly, don't waste your users' ink. For example, white text on an all-black page is probably not the best choice.
